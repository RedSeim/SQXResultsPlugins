📚 Custom Plugin - Sample

Template and guide for creating custom ResultsPlugins

📚 How to Create Your Own Custom Plugin

Quick Start

Creating a custom ResultsPlugin is very simple:

  1. Create a new folder in user/extend/ResultsPlugins/ with your plugin name (e.g., MyCustomPlugin)
  2. Create an index.html file inside that folder
  3. Restart StrategyQuant X
  4. Your plugin will appear as a new tab in the Results section!

How It Works

StrategyQuant X automatically scans the user/extend/ResultsPlugins/ folder and registers each subfolder containing an index.html file as a new tab. Your HTML file is displayed in an iframe, and you can communicate with the parent application using PostMessage API.

What Happens When You Select a Strategy

When a strategy is selected in StrategyQuant X Results section, your plugin will automatically receive:

  • Strategy information (name, project, result key, etc.) via STRATEGY_DATA message
  • You can then request statistics or orders data using PostMessage API

Folder Structure

Your plugin folder structure should look like this:

user/extend/ResultsPlugins/
  └── MyCustomPlugin/
      └── index.html

Available Messages

The plugin uses PostMessage API for communication:

Incoming Messages (from parent window):

  • STRATEGY_DATA - Contains strategy information when a strategy is selected
  • SET_THEME - Contains theme information ('dark' or 'light') when skin is changed or plugin loads
  • STATS_RESPONSE - Contains statistics data after requesting stats
  • ORDERS_RESPONSE - Contains orders data after requesting orders
  • LAST_SETTINGS_XML_RESPONSE - Contains last settings XML data after requesting last settings XML
  • SOURCE_CODE_RESPONSE - Contains generated source code after requesting with GET_SOURCE_CODE
  • SYMBOL_INFO_RESPONSE - Contains symbol info after requesting with GET_SYMBOL_INFO

Outgoing Messages (to parent window):

  • GET_STATS - Request statistics data with parameters
  • GET_ORDERS - Request orders data with parameters
  • GET_LAST_SETTINGS_XML - Request last settings XML containing backtest configuration. Each settings tab in SQX has its own XML section (Data, Money Management, Trading Options, etc.)
  • GET_SOURCE_CODE - Request strategy source code in a given format (xml, pseudo, mq4, mq5, el, etc.)
  • GET_SYMBOL_INFO - Request symbol info

Usage Examples

Example 1: Request Statistics

window.parent.postMessage({
    type: 'GET_STATS',
    params: {
        resultKey: 'Portfolio',
        direction: '0',      // 0=Both, 1=Long, -1=Short
        sampleType: '127',   // 127=Full, 10=In-Sample, 20=Out-of-Sample
        plType: '10'         // 10=Money, 20=Percent, 30=Pips
    }
}, '*');
// Note: projectName, databankName, and strategyName are automatically added from the selected strategy

Example 2: Listen for Strategy Data

window.addEventListener('message', function(event) {
    if (event.data.type === 'STRATEGY_DATA') {
        var strategyName = event.data.data.strategyName;
        var projectName = event.data.data.projectName;
        // Use the data...
    }
});

Example 3: Listen for Stats Response

window.addEventListener('message', function(event) {
    if (event.data.type === 'STATS_RESPONSE') {
        var stats = event.data.data.stats;
        var netProfit = stats.NetProfit;
        var numTrades = stats.NumberOfTrades;
        // Display the stats...
    }
});

Example 4: Request Orders

window.parent.postMessage({
    type: 'GET_ORDERS',
    params: {
        resultKey: 'Portfolio',
        direction: '0',      // 0=Both, 1=Long, -1=Short
        sampleType: '127'    // 127=Full, 10=In-Sample, 20=Out-of-Sample
    }
}, '*');
// Note: projectName, databankName, and strategyName are automatically added from the selected strategy

Example 5: Listen for Theme Changes

window.addEventListener('message', function(event) {
    if (event.data.type === 'SET_THEME') {
        var theme = event.data.theme; // 'dark' or 'light'
        if (theme === 'dark') {
            document.body.style.backgroundColor = '#1a1a1a';
            document.body.style.color = '#ffffff';
        } else {
            document.body.style.backgroundColor = '#F6F6F6';
            document.body.style.color = '#333';
        }
    }
});

Example 6: Request Last Settings XML

window.parent.postMessage({
    type: 'GET_LAST_SETTINGS_XML',
    params: {}
}, '*');
// Note: projectName, databankName, and strategyName are automatically added from the selected strategy

Example 7: Listen for Last Settings XML Response

window.addEventListener('message', function(event) {
    if (event.data.type === 'LAST_SETTINGS_XML_RESPONSE') {
        var lastSettingsXml = event.data.data.lastSettingsXml;
        // lastSettingsXml is a string containing the XML settings from the last backtest
        // Each settings tab in SQX has its own XML section stored:
        // - Data settings (symbol, date range, sample type, etc.)
        // - Money Management settings
        // - Trading Options
        // - And other settings tabs configuration
        // Parse the XML to access specific sections as needed
        // Use the XML data...
    }
});

Example 8: Request Source Code (XML, MT4, MT5, EasyLanguage, etc.)

// Request by format key (same as on Source Code tab):
window.parent.postMessage({
    type: 'GET_SOURCE_CODE',
    params: {
        format: 'xml'   // 'xml' | 'pseudo' | 'mq4' | 'mq5' | 'el' | 'pla' | 'java'
    }
}, '*');

// Or pass full type name:
window.parent.postMessage({
    type: 'GET_SOURCE_CODE',
    params: {
        type: 'Strategy XML'   // e.g. 'MetaTrader 4 (*.mq4)', 'Pseudo Code(*.TXT)'
    }
}, '*');
// Note: projectName, databankName, strategyName are automatically added from the selected strategy

Example 9: Listen for Source Code Response

window.addEventListener('message', function(event) {
    if (event.data.type === 'SOURCE_CODE_RESPONSE') {
        var data = event.data.data;
        var code = data.code;           // generated source code string
        var warning = data.warning;     // optional warning message
        var learnMoreURL = data.learnMoreURL;  // optional link
        if (data.success === 'ok') {
            // Use code (e.g. display in textarea, copy to clipboard)
        }
    }
});

Example 10: Request Symbol Info (GET_SYMBOL_INFO)

window.parent.postMessage({
    type: 'GET_SYMBOL_INFO',
    symbol: 'EURUSD_M1_dukas'
}, '*');
// Response: SYMBOL_INFO_RESPONSE with event.data.data = symbol object or null

Example 11: Listen for Symbol Info Response

window.addEventListener('message', function(event) {
    if (event.data.type === 'SYMBOL_INFO_RESPONSE') {
        var symbolObj = event.data.data;  // the symbol object or null
    }
});

Example 12: Listen for Orders Response

window.addEventListener('message', function(event) {
    if (event.data.type === 'ORDERS_RESPONSE') {
        var orders = event.data.data.orders;
        // orders is an array of order objects
        orders.forEach(function(order) {
            var ticket = order.Ticket;
            var pl = order.PL;
            var openTime = new Date(order.OpenTime);
            // Order.Type is a byte constant: 1=Buy, 2=Sell, 3=BuyLimit, 4=SellLimit, 5=BuyStop, 6=SellStop
            var orderType = order.Type;
            // Convert to string if needed:
            var orderTypeString = getOrderTypeString(orderType);
            // Order.CloseType is a byte constant: 1=Manual, 2=SL, 3=PT, 4=EndTest, etc.
            var closeType = order.CloseType;
            // Convert to string if needed:
            var closeTypeString = getOrderCloseTypeString(closeType);
            // Process each order...
        });
    }
});

// Helper function to convert order type constant to string
function getOrderTypeString(type) {
    switch(type) {
        case 0: return 'Any';
        case 1: return 'Buy';
        case 2: return 'Sell';
        case 3: return 'BuyLimit';
        case 4: return 'SellLimit';
        case 5: return 'BuyStop';
        case 6: return 'SellStop';
        case 7: return 'BuyStopLimit';
        case 8: return 'SellStopLimit';
        case 9: return 'Deposit';
        case 10: return 'Withdrawal';
        case 11: return 'Balance';
        default: return 'Unknown';
    }
}

// Helper function to convert order close type constant to string
function getOrderCloseTypeString(closeType) {
    switch(closeType) {
        case 1: return 'Manual';
        case 2: return 'SL';
        case 3: return 'PT';
        case 4: return 'EndTest';
        case 5: return 'End Of Day';
        case 6: return 'Expired';
        case 7: return 'Reversed';
        case 8: return 'Deleted';
        case 9: return 'Replaced';
        case 11: return 'OCA';
        case 12: return 'Commission';
        case 13: return 'End Of Day (Time)';
        case 14: return 'End Of Friday';
        case 16: return 'End Of Friday (Time)';
        case 17: return 'End Of Range';
        case 18: return 'Control order';
        case 19: return 'Exit After X Bars';
        case 20: return 'MoveSL2BE';
        case 21: return 'TrailingStop';
        case 22: return 'Exit Signal';
        case 55: return 'End Of Day (Next Market Open)';
        case 60: return 'Delisted';
        default: return 'Unknown';
    }
}

Available Statistics Fields

When you receive STATS_RESPONSE, the stats object contains:

  • NetProfit - Net profit in currency
  • NetProfitInPips - Net profit in pips
  • NumberOfTrades - Total number of trades
  • Drawdown - Maximum drawdown (in currency, percent, or pips depending on plType)
  • PctDrawdown - Maximum drawdown percentage
  • PipsDrawdown - Maximum drawdown in pips
  • AvgProfitByDay - Average profit per day
  • AvgProfitByMonth - Average profit per month
  • AvgProfitByYear - Average profit per year
  • AvgPctProfitByYear - Average percentage profit per year
  • AvgTrade - Average profit per trade
  • AvgAbsTrade - Average absolute profit per trade
  • AvgTradesPerDay - Average number of trades per day
  • AvgTradesPerMonth - Average number of trades per month
  • AvgTradesPerYear - Average number of trades per year
  • ProfitFactor - Profit factor (Gross Profit / Gross Loss)
  • SharpeRatio - Sharpe ratio (risk-adjusted return measure)
  • Stability - Stability metric (how straight and steep is the equity curve)
  • Symmetry - Symmetry between Long and Short side profits
  • ReturnDDRatio - Return to Drawdown ratio
  • ReturnOpenDDRatio - Return to Open Drawdown ratio
  • AAR_DD_Ratio - Average Annual Return to Drawdown ratio
  • CalmarRatio - Calmar ratio (return to max drawdown)
  • RExpectancy - Risk-adjusted expectancy
  • RExpectancyScore - Risk-adjusted expectancy score
  • SQN - System Quality Number
  • SQNScore - SQN Score (as decimal, e.g., 0.3 = 30%)
  • WinLossRatio - Win/Loss ratio
  • WinningPct - Winning percentage
  • PayoutRatio - Profit factor (Payout Ratio)
  • AvgBarsTrade - Average number of bars per trade
  • AHPR - Average Holding Period Return
  • ZScore - Z-Score statistical measure
  • ZProbability - Z-Score probability
  • Expectancy - Expected value per trade
  • StandardDev - Standard deviation
  • Exposure - Market exposure percentage
  • ExposurePosition - Position exposure
  • StagnationPeriod - Stagnation period (number of bars)
  • StagnationPeriodPct - Stagnation period percentage
  • StagnationFrom - Stagnation period start (bar index)
  • StagnationTo - Stagnation period end (bar index)
  • DegreesOfFreedom - Degrees of freedom
  • TotalTradingDays - Total number of trading days
  • TotalTradingMonths - Total number of trading months
  • TotalTradingYears - Total number of trading years
  • ProfitableMonths - Number of profitable months
  • NumberOfProfits - Number of winning trades
  • NumberOfLosses - Number of losing trades
  • NumberOfCanceled - Number of canceled trades
  • GrossProfit - Total gross profit
  • GrossLoss - Total gross loss
  • AvgWin - Average winning trade
  • AvgLoss - Average losing trade
  • MaxProfit - Maximum profit in a single trade
  • MaxLoss - Maximum loss in a single trade
  • MaxConsecWins - Maximum consecutive wins
  • MaxConsecLoss - Maximum consecutive losses
  • AvgConsecWin - Average consecutive wins
  • AvgConsecLoss - Average consecutive losses
  • AvgBarsWin - Average bars in winning trades
  • AvgBarsLoss - Average bars in losing trades
  • Commission - Total commission costs
  • SlippageInMoney - Total slippage costs in currency
  • CommSwapInMoney - Total commission and swap costs in currency
  • MaxNewHighDurationFrom - Max new high duration start (bar index)
  • MaxNewHighDurationTo - Max new high duration end (bar index)
  • MaxNewHighDurationPeriod - Max new high duration period (bars)
  • MaxNewHighDurationPct - Max new high duration percentage

Available Order Fields

When you receive ORDERS_RESPONSE, the orders array contains objects with:

  • Ticket - Order ticket number
  • Symbol - Trading symbol
  • ResultName - Setup/result name
  • NumberOfTrade - Order number in sequence
  • OpenTime - Order open time (timestamp)
  • CloseTime - Order close time (timestamp)
  • OriginalOpenTime - Original order open time (for pending orders)
  • Type - Order type (byte constant: 0=Any, 1=Buy, 2=Sell, 3=BuyLimit, 4=SellLimit, 5=BuyStop, 6=SellStop, 7=BuyStopLimit, 8=SellStopLimit, 9=Deposit, 10=Withdrawal, 11=Balance, 100=BuyToCoverStop, 101=BuyToCoverLimit, 102=SellToCoverStop, 103=SellToCoverLimit)
  • CloseType - How the order was closed (byte constant: 1=Manual, 2=SL, 3=PT, 4=EndTest, 5=End Of Day, 6=Expired, 7=Reversed, 8=Deleted, 9=Replaced, 11=OCA, 12=Commission, 13=End Of Day (Time), 14=End Of Friday, 16=End Of Friday (Time), 17=End Of Range, 18=Control order, 19=Exit After X Bars, 20=MoveSL2BE, 21=TrailingStop, 22=Exit Signal, 55=End Of Day (Next Market Open), 60=Delisted)
  • SampleType - Sample type (byte constant: 127=Full, 10=In-Sample, 20=Out-of-Sample)
  • OpenPrice - Order open price
  • ClosePrice - Order close price
  • OriginalOpenPrice - Original order open price (for pending orders)
  • SLLevel - Stop Loss price level
  • PTLevel - Profit Target price level
  • Size - Order size (lot size)
  • BarsInTrade - Number of bars the order was open
  • TimeInTrade - Duration in seconds
  • ProfitLoss - Profit/Loss in currency (same as PL)
  • ProfitLossPct - Profit/Loss as percentage
  • ProfitLossPips - Profit/Loss in pips
  • Drawdown - Drawdown at order close (same as DD)
  • PctDrawdown - Drawdown as percentage
  • MAE - Maximum Adverse Excursion (in currency)
  • MAEPips - Maximum Adverse Excursion in pips
  • MFE - Maximum Favorable Excursion (in currency)
  • MFEPips - Maximum Favorable Excursion in pips
  • Comment - Order comment
  • CommSwap - Commission and swap costs
  • SlippageInMoney - Slippage in currency
  • Balance - Account balance after order close

Note: Some fields may be null or have default values (e.g., NOT_DEFINED = -99999999) if not applicable to the order.
